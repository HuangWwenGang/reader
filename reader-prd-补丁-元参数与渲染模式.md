# 阅读器 PRD 补丁：元参数 & 渲染模式

> 这是针对当前开发进度的增补，配合主 PRD 使用。
> 当前状态：UI 基本完成、划线基本可用、能正常导入并阅读 EPUB。
> 本补丁解决两件事：(1) 明确需要哪些元参数；(2) 给"无限流漂移/卡顿"一个明确处置。

---

## A. 渲染模式的取舍（先解决痛点）

### 现状
- 分页翻页模式：稳定，无明显问题。
- 无限流（连续滚动）模式：章节跳转有位置漂移，且有卡顿。

### 结论与要求
- **第一版默认使用分页翻页模式。** 将无限流标记为实验性 / 默认关闭。
- 原因：漂移和卡顿属于"会破坏阅读"的硬伤，而滚动顺滑只是"锦上添花"。在核心功能（划线/小结/复习）跑通前，不值得为无限流投入虚拟滚动这类高成本优化。
- **不要求**在第一版修复无限流。把它作为已知限制记录在案，留待后续。
- 如果未来要优化无限流，正确方向是**虚拟滚动**（只渲染视口附近内容、动态回收远处 DOM、为未加载内容预留占位高度以减少跳转漂移），但这是独立的优化任务，不属于第一版。

## B. 书籍元数据（Book Metadata）

从 EPUB 文件中解析并存储以下字段。EPUB 的元数据在 `content.opf` 里，foliate-js 解析后通常可直接读取。**取不到的字段一律允许为空，不要因为缺字段而报错或阻断导入。**

必备（能取就取，取不到留空）：
- `title` 书名
- `author` 作者
- `cover` 封面图（Blob；取不到用纯色 + 书名占位）
- `language` 语言（用于排版方向、字体等，例如 zh / en / ja）

可选（有则存，无则忽略，不强求）：
- `publisher` 出版社
- `publishDate` 出版日期
- `identifier` 书的唯一标识（EPUB 里的 ISBN 或 UUID，可用于判断"这本书是否已导入过"，避免重复导入）
- `description` 简介
- `totalChapters` 章节总数（来自 TOC）

> 注意：EPUB 格式不标准，很多书缺字段是常态。解析要"尽力而为、缺了不崩"。对中文 EPUB 尤其要测试 `title`/`author` 是否有乱码或编码问题。

## C. 阅读状态参数（Reading State）—— 关于"页码"的关键说明

**这里有个 EPUB 的本质问题必须讲清楚：EPUB 没有固定页码。**

EPUB 是回流（reflowable）格式，页码取决于字号、屏幕尺寸、行距——同一本书在不同设置下"页数"完全不同。所以不要试图给 EPUB 一个像纸书那样固定的"第 X 页 / 共 Y 页"，那是做不准的。

正确的做法是用以下参数替代传统页码：

必备：
- `currentLocation` 当前阅读位置（foliate-js 的定位标识，通常是 CFI 或等价值）—— 用于"重开回到原位"，这是最核心的一个，必须存。
- `progressPercent` 全书阅读进度百分比（0–100%）—— 这是给 EPUB 的"页码替代品"，用百分比表达"读到全书的哪里"。foliate-js 一般能提供。
- `currentChapterTitle` 当前所在章节标题 —— 比页码更有意义的位置感（用户更关心"我在第几章"而不是"第几页"）。

可选（增强体验）：
- `chapterProgressPercent` 当前章节内的进度百分比
- `estimatedPageInChapter` / `estimatedTotalPagesInChapter` 当前显示设置下、本章内的"动态页码"（仅在分页模式下有意义，且会随字号变化，仅供参考显示，不要持久化为书的固有属性）

> 一句话原则：**用"百分比 + 章节标题 + CFI 位置"这套组合，取代纸书的固定页码。** 这是 EPUB 阅读器的标准解法。如果一定要显示页码，只在分页模式下显示"当前设置下的动态页码"，并明确它会随字号变化。

## D. 阅读时长 / 行为数据（为后续复习和报告埋的燃料）

主 PRD 提到后续要做复习和报告，这些功能需要数据燃料。第一版**只需老实记录、存下来即可，不需要展示或分析**：

- `readingSessions` 每次阅读的记录：开始时间、结束时间、本次读了哪些章节 / 进度区间。
- 每本书的 `totalReadingTime` 累计阅读时长。
- 每次阅读触及的章节范围（供后续"脉络回忆"和"AI 上下文"使用）。

> 这些是后续 V2（复习、AI 问答、报告）的原料。现在不存，将来这些功能就无米下炊。存储成本极低，第一版就该埋下。

---

## E. 数据模型补充（在主 PRD 基础上增补字段）

```
Book {
  id, title, author, cover, fileBlob, createdAt   // 主 PRD 已有
  // 新增元数据：
  language?, publisher?, publishDate?, identifier?, description?, totalChapters?
  // 新增阅读状态：
  currentLocation?      // CFI，重开回到原位
  progressPercent?      // 全书进度 0-100
  currentChapterTitle?  // 当前章节标题
  totalReadingTime?     // 累计阅读时长（秒）
}

ReadingSession {        // 新增表：为后续复习/报告埋的数据
  id, bookId
  startedAt, endedAt
  chapterRange          // 本次读了哪些章节/进度区间
}
```

Highlight 表维持主 PRD 不变。

## F. 验收补充

在主 PRD 验收标准基础上，增加：
1. 导入中文 EPUB 后，书名/作者显示正常无乱码（取不到则合理留空，不崩溃）。
2. 阅读界面能显示"全书进度百分比 + 当前章节标题"，作为页码替代。
3. 关闭重开后回到上次精确位置（currentLocation 生效）。
4. 默认是分页翻页模式，稳定无漂移；无限流若保留则为实验性、默认关闭。
5. 每次阅读的时长和章节范围被记录进 ReadingSession（即便界面暂不展示）。

# 阅读器 V1 产品需求文档（PRD）

> 给开发者（Claude Code）的说明：这是一个**个人自用**的开源阅读工具的第一版需求。
> 第一版的唯一目标是：**让用户能把一本 EPUB 读进去、划线、在划线的当下原地写下想法，并且数据存得住、关掉再打开还在。**
> 请严格按本文档实现，**不要**实现"非目标"章节里列出的任何功能。如果对某处有歧义，倾向于做最简单、最不容易出错的实现，并在代码注释里标注该处的假设。

---

## 1. 背景与动机

用户日常用 iOS 自带 Books 阅读 EPUB。痛点是：读书时产生的思考，因为"要切换到别的 App 去记录"这一步的麻烦，常常就懒得记了，导致读完一本书脑袋空空、思考全部遗忘。

核心洞察：**记录的物理成本越接近于零，用户越愿意记。** 因此本工具的命根子是——在用户划线的那一刻，**原地**弹出输入框，让他不离开阅读界面就能写下想法。

本工具是 Web（PWA）应用，不是原生 App。理由：免去 Mac/Xcode/签名/审核的全部负担，改代码即时生效，开源后他人打开网址即可用。

---

## 2. 目标用户

就是开发者本人（单用户、自用）。因此：
- **无需**用户系统、登录、注册、账号。
- **无需**多端同步（第一版数据存在单一设备本地即可）。
- 不必考虑高并发、不必考虑陌生用户引导。优先保证"我自己用着顺手"。

---

## 3. 第一版范围（必须实现）

### 3.1 导入与阅读
- 用户可从本地选择一个 `.epub` 文件导入。
- 使用 **foliate-js**（https://github.com/johnfactotum/foliate-js ）渲染 EPUB，支持：
  - 正常分页翻页（点击左右区域或滑动翻页均可，优先实现点击翻页）
  - 目录（TOC）侧边栏，点击可跳转章节
  - 记住上次读到的位置（reading location / CFI），重新打开同一本书时回到该位置
- PDF 暂**不要求**在第一版实现（见非目标）。第一版只做 EPUB。

### 3.2 书架
- 一个简单的书架页，列出已导入的所有书（封面 + 书名即可，封面取不到就用纯色占位）。
- 点击某本书进入阅读。
- 能删除某本书（连带其划线数据一起删，删除前给个确认）。

### 3.3 划线（核心）
- 用户在阅读时可以**选中一段文字**，触发划线。
- 划线后，该段文字在原文里以高亮样式持久显示（重开后依然高亮）。
- 划线需要记录其在书中的精确定位（foliate-js 提供的 CFI 或等价定位信息），以便重开后准确还原高亮位置。

### 3.4 划线弹窗 —— 本工具的命根子，交互必须丝滑
- 用户完成划线的**当下**，在选区附近**原地弹出**一个轻量输入框（不要跳转新页面、不要全屏遮罩到看不见原文）。
- 弹窗内包含：
  1. 一个多行文本输入框：写下"我对这段的想法/为什么划它"。
  2. 一个**可选**的分类标签选择（预设几个标签即可，例如：`金句` `疑问` `启发` `反对` `待查`；允许不选）。
  3. 保存 / 取消两个动作。取消则只保留高亮、不保留想法；保存则高亮+想法+标签一起存。
- 交互要求（重要）：
  - 弹出要快，输入框自动获得焦点，用户划完线手不用动就能直接打字。
  - 不打断阅读：弹窗关闭后立刻回到原阅读位置，不跳页、不丢失滚动位置。
  - 想法**可以为空**也能保存（有些划线就是单纯划个高亮）。允许之后再补写。
- 已有划线可以**重新打开编辑**它的想法和标签。

### 3.5 笔记列表（单本书维度）
- 在阅读界面提供一个入口，查看**当前这本书**的所有划线 + 想法列表（按在书中出现的先后排序）。
- 点击某条笔记，能跳回原文对应位置。

### 3.6 数据持久化
- 全部数据存在浏览器本地 **IndexedDB**：书籍文件（或其引用）、阅读进度、划线、想法、标签。
- 关闭浏览器/刷新后数据不丢失。
- 提供一个"**导出全部数据为 JSON**"的按钮（用户的笔记是他的核心资产，必须能导出备份；这也为将来的复习/报告功能预留了数据出口）。第一版可不做导入 JSON，但导出必须有。

### 3.7 PWA
- 配置 manifest 与 service worker，使其可"添加到主屏幕"后**全屏运行**（无浏览器地址栏），有应用图标。
- 支持离线打开已导入的书（service worker 缓存应用壳；书数据本就在 IndexedDB）。

## 4. 非目标（第一版明确不做，请勿实现）

以下功能属于后续版本，**第一版一律不要实现，连脚手架/占位入口都不要加**，以免增加复杂度和出错面：

- ❌ 任何大语言模型 / AI 相关功能（AI 帮读、AI 提问、AI 辅助写想法等）—— 留给 V2
- ❌ 间隔复习 / 复习提醒 / 记忆曲线 —— 留给 V3
- ❌ 爬取或展示他人书评 —— 留给后续
- ❌ 读后总结流程 —— 留给后续
- ❌ 年度报告 / 分享海报 —— 留给最后
- ❌ 用户账号、登录、云同步、后端服务器
- ❌ PDF 阅读（第一版只做 EPUB）
- ❌ 多端数据同步

> 设计原则：上述功能越靠后越依赖"前面已积累的真实笔记数据"。在没有数据之前实现它们等于给空房间装修。

---

## 5. 技术栈（已锁定，请勿替换）

- **框架**：React + Vite（TypeScript 优先；若显著增加复杂度可用 JS）
- **EPUB 渲染**：foliate-js
- **本地存储**：IndexedDB（可用 `idb` 这个轻量封装库简化操作）
- **PWA**：vite-plugin-pwa 或手写 manifest + service worker
- **样式**：任选（Tailwind 或普通 CSS Module 均可），但 UI 要干净克制、以阅读为中心，不要花哨
- **后端**：无。第一版纯前端、纯本地。

---

## 6. 数据模型（建议，可微调）

```
Book {
  id: string            // uuid
  title: string
  author?: string
  cover?: Blob | null
  fileBlob: Blob        // epub 文件本体
  lastLocation?: string // foliate-js 的定位信息，用于恢复阅读位置
  createdAt: number
}

Highlight {
  id: string            // uuid
  bookId: string        // 关联 Book
  cfi: string           // 在书中的精确定位
  text: string          // 被划线的原文文字
  note?: string         // 用户写的想法，可为空
  tag?: string          // 分类标签，可为空
  createdAt: number
  updatedAt: number
}
```

## 7. 验收标准（第一版做完后，下列每条都应成立）

1. 能导入一个真实 EPUB 并正常翻页阅读，目录可跳转。
2. 关闭后重开，能回到上次阅读位置。
3. 选中文字能划线，划线后原地弹出输入框、自动聚焦、可立即打字。
4. 写下想法+选标签保存后，刷新页面，高亮和想法都还在。
5. 想法为空也能保存（仅高亮）。
6. 已有划线能重新打开编辑想法。
7. 能查看当前书的全部笔记列表并跳回原文。
8. 能删除一本书。
9. 能导出全部数据为 JSON 文件。
10. 添加到 iOS 主屏后全屏运行、可离线打开已导入的书。

---

## 8. 给开发者的优先级提示

如果时间/篇幅有限，按此顺序保证质量：
1. EPUB 能读 + 翻页 + 记住位置（基础阅读体验，foliate-js 负责大部分）
2. **划线 + 原地弹窗写想法 + 持久化**（这是命根子，宁可其他地方简陋，这里必须丝滑）
3. 笔记列表 + 跳回原文
4. 数据导出
5. PWA 全屏 / 离线
6. 书架管理（多本书、删除）

第 2 点的体验质量 = 整个产品的成败。请在这里多花心思。
